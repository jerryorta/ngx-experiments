Crosshair — Usage

The shared crosshair snaps a dashed vertical guide to the nearest datum x, marks each series with a focus dot, and — with shared: true — shows ONE tooltip listing every series' value at that x. The shared tooltip is a real Angular tooltip: the chart runs [chromelessTooltip]="true" and each <nge-chart> projects a #ngeChartTooltip template that renders the card from content.rows (a swatch row.color, row.label, and row.value per series). Move the pointer across a plot to try it.

1. Line host — multi-series

Three series on a LINE host. Enable the crosshair on the chart's base.crosshair, then render the shared card from content.rows in a chromeless #ngeChartTooltip template. 

const cfg = createLineChartConfig({{ "{" }} data, seriesColors {{ "}" }});
cfg.base.crosshair = {{ "{" }} x: true, shared: true {{ "}" }};

<nge-chart [config]="cfg" [chromelessTooltip]="true">
  <ng-template #ngeChartTooltip let-content>
    <!-- header = content.label, one row per content.rows entry -->
  </ng-template>
</nge-chart>
{{ content?.label }}
@for (row of rowsOf(content); track row.label) {
{{ row.label }} {{ row.value }}
}

2. Area host — multi-series

The SAME crosshair over an AREA host (overlaid fills with a top stroke). The engine-level crosshair sources its rows from any line/area layer, so nothing about the tooltip wiring changes between host types.

{{ content?.label }}
@for (row of rowsOf(content); track row.label) {
{{ row.label }} {{ row.value }}
}

3. Single series

With one series the shared tooltip degrades gracefully to a single row (labelled Value when the data carries no seriesId). The guide + focus dot behave exactly as in the multi-series case.

{{ content?.label }}
@for (row of rowsOf(content); track row.label) {
{{ row.label }} {{ row.value }}
}
