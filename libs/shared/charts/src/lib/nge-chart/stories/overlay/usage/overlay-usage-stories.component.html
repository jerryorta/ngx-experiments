Overlay Layer - Usage Examples

The overlay layer draws an analytical annotation OVER a line / scatter host on the host's shared scales. It is composed with addOverlay() (or by dropping createOverlayConfig() into a layers array) and fans out across three modes — trendline, control, and fan. Every host below uses a numeric x so its scale is continuous (trendline and fan require an invertible x-scale).

1. Composing a Trend Line (addOverlay)

addOverlay() appends the overlay as the last (front) layer of an existing config. Here a least-squares fit: 'linear' trend line is drawn over the host line, sharing its scales. 

const base = createLineChartConfig({{ "{" }}
  data,
  showPoints: true,
  showXAxis: true,
  showYAxis: true,
{{ "}" }});

const config = addOverlay(base, {{ "{" }}
  data,          // the SAME x/y series the host draws
  fit: 'linear',
  mode: 'trendline',
{{ "}" }});

<nge-chart [config]="config" />

2. Trend Fit: Linear vs LOESS

fit: 'linear' strokes a straight least-squares line; fit: 'loess' strokes a smooth local-regression curve that follows local structure (tune it with loessBandwidth).

fit: 'linear'
fit: 'loess'

3. Control Chart (mean + ±3σ limits + band)

mode: 'control' draws the series mean plus symmetric ±sigma·σ control limits. Set showControlBand: true to shade the in-control zone between the limits. Give the host a yDomain that contains the limits so nothing clips. 

const config = addOverlay(
  createLineChartConfig({{ "{" }} data, showPoints: true, yDomain: [20, 80] {{ "}" }}),
  {{ "{" }}
    data,
    mode: 'control',
    showControlBand: true,
    sigma: 3,
  {{ "}" }},
);

4. Control-Limit Width — 2σ vs 3σ

sigma sets the limit half-width in standard deviations. A tighter 2σ envelope flags more points as out-of-control than the classic 3σ chart.

sigma: 2
sigma: 3

5. Fan Chart (prediction intervals)

mode: 'fan' draws one widening prediction-interval band per intervals level. The bands fan out toward the forecast horizon — a higher level yields a wider band. Fan mode needs a continuous (invertible) x. 

const config = addOverlay(
  createLineChartConfig({{ "{" }} data, showPoints: true, yDomain: [0, 115] {{ "}" }}),
  {{ "{" }}
    data,
    intervals: [0.5, 0.8, 0.95],
    mode: 'fan',
  {{ "}" }},
);

6. Multi-Series Trend (one fit per series)

When the overlay data carries a seriesId, trendline mode fits and strokes one reference line PER series. Here two store series each get their own linear trend.

7. createOverlayConfig in a Layers Array

addOverlay() is a shorthand — under the hood the overlay is just another entry in layers. Use createOverlayConfig() directly when you are assembling a multi-layer config by hand. 

const base = createLineChartConfig({{ "{" }} data, showPoints: true {{ "}" }});

const config = {{ "{" }}
  ...base,
  layers: [
    ...base.layers,
    createOverlayConfig({{ "{" }} data, fit: 'loess', mode: 'trendline' {{ "}" }}),
  ],
{{ "}" }};

8. Dynamic Data with Signals

Recommended pattern: hold the series in a signal and rebuild the composed config in a computed. The host line and its trend overlay animate together when the data changes. 

data = signal<NgeLineDataPoint[]>([...]);

config = computed(() =>
  addOverlay(
    createLineChartConfig({{ "{" }} data: this.data(), showPoints: true {{ "}" }}),
    {{ "{" }} data: this.data(), fit: 'linear', mode: 'trendline' {{ "}" }},
  )
);
