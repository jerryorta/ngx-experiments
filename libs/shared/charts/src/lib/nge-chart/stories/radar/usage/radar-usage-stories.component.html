Radar - Usage Examples

Practical examples for the radar / polar (spider / star) layer. The unique dimension labels become evenly-angled spokes (first straight up), and each seriesId draws a closed polygon whose vertex radius encodes value. render picks the shape: 'area' fills the polygon (Radar Diagram), 'line' strokes the outline only (Polar Chart). Geometry is self-computed from the container (center + radius) — the cartesian axes stay off.

1. Basic Radar (single series)

Simplest way to create a radar using createRadarChartConfig(). One value per attribute axis draws a single filled polygon over the spoke web. 

data = [
  {{ "{" }} label: 'Speed', value: 80 {{ "}" }},
  {{ "{" }} label: 'Power', value: 65 {{ "}" }},
  // ...
];

config = createRadarChartConfig({{ "{" }}
  data: this.data,
{{ "}" }});

<nge-chart [config]="config" />

2. Multi-series Comparison

Group points by seriesId to overlay several profiles on the same axes — one closed polygon per series, colored from the palette by series index. Ideal for comparing products / candidates across shared metrics. 

data = [
  {{ "{" }} label: 'Speed', seriesId: 'Falcon', value: 80 {{ "}" }},
  {{ "{" }} label: 'Speed', seriesId: 'Vortex', value: 55 {{ "}" }},
  // ...
];

config = createRadarChartConfig({{ "{" }}
  data: this.data,
  tooltip: {{ "{" }} enabled: true {{ "}" }},
{{ "}" }});

3. Polar Chart (render: 'line')

render: 'line' strokes the series outline only — no fill — with small vertex dots. The same radial axes read as a polar chart, cleaner when several series overlap. 

config = createRadarChartConfig({{ "{" }}
  data: this.data,
  render: 'line', // outline only, no fill
{{ "}" }});

4. Grid Levels

levels sets how many concentric value rings the grid web draws. Unset, the rings fall on the radial scale's own ticks; a fixed levels pins the count. 

config = createRadarChartConfig({{ "{" }}
  data: this.data,
  levels: 3, // three concentric rings
{{ "}" }});

5. Inner Hub (innerRadius ratio)

innerRadius is a ratio (0–1) of the self-computed outer radius, not pixels — so it stays resize-safe. A small value lifts the axes off a center hub so low values near the middle stay readable. 

config = createRadarChartConfig({{ "{" }}
  data: this.data,
  innerRadius: 0.15, // center hub
{{ "}" }});

6. Click Handling

Provide an onClick callback in the config. The event carries the full data point for the hovered vertex (series + dimension + value), so you can react to a selection. 

config = createRadarChartConfig({{ "{" }}
  data: this.data,
  onClick: (event) => {{ "{" }}
    console.log('Clicked:', event.data.seriesId, event.data.label, event.data.value);
  {{ "}" }},
{{ "}" }});
Last clicked: {{ lastClicked() }}

7. Dynamic Data with Signals

Drive the chart from a signal and rebuild the config with computed(). Re-rolling every value re-runs the radial scale — the polygon grows / shrinks to its new shape, animated. 

dynamicData = signal<NgeRadarDataPoint[]>(this.data);

dynamicConfig = computed(() =>
  createRadarChartConfig({{ "{" }} data: this.dynamicData() {{ "}" }})
);

<nge-chart [config]="dynamicConfig()" />
