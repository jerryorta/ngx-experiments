Sunburst Chart - Usage Examples

Practical examples for the sunburst / icicle layer. One hierarchical primitive partitions a nested {{ "{" }} label, value?, children? {{ "}" }} tree three ways via config: a full sunburst (innerRadius: 0), a multi-level donut (innerRadius as a ratio 0–1 carves the center hole), and an icicle (layout: 'linear' draws stacked columns). Leaves carry a value; internal nodes omit it and are summed from their children.

1. Basic Sunburst

Simplest way to create a sunburst using createSunburstChartConfig(). Provide a nested array of NgeHierarchyDatum — one ring per depth level, one arc per node, in input order. 

data: NgeHierarchyDatum[] = [
  {{ "{" }}
    label: 'Housing',
    children: [
      {{ "{" }} label: 'Rent', value: 1800 {{ "}" }},
      {{ "{" }} label: 'Utilities', children: [ /* Electric, Water, Internet */ ] {{ "}" }},
      {{ "{" }} label: 'Insurance', value: 150 {{ "}" }},
    ],
  {{ "}" }},
  // Food, Transport ...
];

config = createSunburstChartConfig({{ "{" }} data: this.data {{ "}" }});

<nge-chart [config]="config" />

2. Multi-level Donut

innerRadius is a ratio (0–1) of the self-computed outer radius, not pixels — so it stays resize-safe. 0.3 carves a center hole, pushing the rings outward for a multi-level donut. 

config = createSunburstChartConfig({{ "{" }}
  data: this.data,
  innerRadius: 0.3, // donut hole
{{ "}" }});

3. Icicle (Linear Layout)

layout: 'linear' swaps the concentric rings for stacked rectangle columns — the same partition, drawn as an icicle. Depth runs left-to-right; each column's height is proportional to its summed value. 

config = createSunburstChartConfig({{ "{" }}
  data: this.data,
  layout: 'linear', // icicle
{{ "}" }});

4. Depth Cap

maxDepth renders at most that many rings / columns — handy for a summary view of a deep tree. Here 2 shows only the top two levels; the grandchildren are folded into their parents. 

config = createSunburstChartConfig({{ "{" }}
  data: this.data,
  innerRadius: 0.3,
  maxDepth: 2, // top two levels only
{{ "}" }});

5. Custom Branch Colors

Pass a seriesColors palette — the top-level branch index maps to colors[index % length], and every descendant inherits its branch color. A per-node color on any datum wins over the palette for that node. 

config = createSunburstChartConfig({{ "{" }}
  data: this.data,
  innerRadius: 0.3,
  seriesColors: ['#1E88E5', '#43A047', '#FB8C00'],
{{ "}" }});

6. Click Handling

Provide an onClick callback in the config. The event carries the full data node for the clicked arc — internal nodes omit value (it is summed by the renderer), so guard for it. 

config = createSunburstChartConfig({{ "{" }}
  data: this.data,
  onClick: (event) => {{ "{" }}
    console.log('Clicked:', event.data.label, event.data.value);
  {{ "}" }},
{{ "}" }});
Last clicked: {{ lastClicked() }}

7. Dynamic Data with Signals

Drive the chart from a signal and rebuild the config with computed(). Re-rolling every leaf's value re-runs the partition — the internal-node sums and arc angles update together, animated. 

dynamicData = signal<NgeHierarchyDatum[]>(this.data);

dynamicConfig = computed(() =>
  createSunburstChartConfig({{ "{" }} data: this.dynamicData(), innerRadius: 0.3 {{ "}" }})
);

<nge-chart [config]="dynamicConfig()" />
