Stacked Bar Chart - Usage Examples

Practical patterns for createStackedBarChartConfig(). Data is long-format — {{ "{" }} category, seriesId, value {{ "}" }} rows where points sharing a category stack into one column and points sharing a seriesId form one series. A single family delivers three catalog types: plain stacked, 100% stacked, and Marimekko.

1. Stacked Bar (stackOffset: 'none')

Segments stack from a zero baseline, so each column's height is the quarter total and every band reads as an absolute contribution. This is the default — omit stackOffset for the same result. 

data = [
  {{ "{" }} category: 'Q1', seriesId: 'Cloud',    value: 120 {{ "}" }},
  {{ "{" }} category: 'Q1', seriesId: 'Licenses', value:  90 {{ "}" }},
  // ...one row per series × quarter
];

config = createStackedBarChartConfig({{ "{" }}
  data,
  legend: {{ "{" }} enabled: true {{ "}" }},
  seriesColors: ['#1E88E5', '#43A047', '#FB8C00', '#8E24AA'],
  showLabels: true,
  stackOffset: 'none',
{{ "}" }});

<nge-chart [config]="config" />

2. 100% Stacked Bar (stackOffset: 'expand')

stackOffset: 'expand' normalises every column to full height, so each band shows its share of the quarter total regardless of the absolute magnitude. 

config = createStackedBarChartConfig({{ "{" }}
  data, // Cloud + Licenses + Services + Support
  stackOffset: 'expand', // each column → 0…100%
{{ "}" }});

3. Marimekko (bandWidthAccessor + expand)

Supply a bandWidthAccessor to make each column's WIDTH proportional to its group total, then pair it with stackOffset: 'expand' for the classic mosaic where BOTH dimensions encode data. Marimekko is always laid out vertically. 

config = createStackedBarChartConfig({{ "{" }}
  data,
  bandWidthAccessor: (_category, total) => total, // width ∝ total
  stackOffset: 'expand',                          // height → share
{{ "}" }});

4. Horizontal Stacked Bar

orientation: 'horizontal' swaps the band and value axes — the stack grows left-to-right. Available for the plain and 100% modes (not Marimekko). 

config = createStackedBarChartConfig({{ "{" }}
  data,
  orientation: 'horizontal',
  stackOffset: 'none',
{{ "}" }});

5. Value Labels + Tooltip

showLabels prints each segment's value in place, and tooltip surfaces the series + category + value on hover. 

config = createStackedBarChartConfig({{ "{" }}
  data,
  showLabels: true,
  tooltip: {{ "{" }}
    enabled: true,
    position: 'follow-mouse',
    width: 160,
  {{ "}" }},
{{ "}" }});

6. Click to Inspect a Segment

Provide an onClick callback — each segment hands back its source datum (category, seriesId, value). 

config = createStackedBarChartConfig({{ "{" }}
  data,
  onClick: (event) => {{ "{" }}
    inspect(event.data.seriesId, event.data.category, event.data.value);
  {{ "}" }},
{{ "}" }});

7. Dynamic Data With Signals

Recommended pattern: hold the data in a signal() and rebuild the config inside a computed(). Re-rolling the values recomputes the config and the segments animate between states. 

liveData = signal<NgeStackedBarDataPoint[]>(buildData());

config = computed(() =>
  createStackedBarChartConfig({{ "{" }} data: liveData(), stackOffset: 'none' {{ "}" }})
);

<nge-chart [config]="config()" />
