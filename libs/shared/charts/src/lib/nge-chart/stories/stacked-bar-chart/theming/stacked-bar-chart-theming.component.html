Stacked Bar Chart - Theming Examples

Default Theme

No overrides — the four stack series pull their colors from the theme's stacked-bar.bar.colors palette (which resolves to var(--chart-*) tokens).

Custom Series Palette (seriesColors)

The seriesColors preset option sets the per-series fill palette and colors the legend swatches to match. Series index maps to colors[index % length].

Palette via theme['stacked-bar'].bar.colors

The same palette can be set on the theme slice instead of seriesColors — useful when you compose a full theme object rather than passing options.

Segment Separators + Radius

theme['stacked-bar'].bar.stroke + strokeWidth carve a gap between adjacent segments, and radius rounds their corners.

Value Label Styling

Color, size, and weight of the per-segment value labels via theme['stacked-bar'].label (with showLabels on).

Custom Axis + Gridlines

Axis label / line / tick colors via theme.axis and gridline color / dash / width via theme.grid (with showYGrid on).

Multiple Charts with Different Palettes

The same stacked dataset rendered with three different seriesColors palettes. Each chart is the same <nge-chart> component with a different config.

Branded via --chart-* Token Overrides (light + dark)

The layer's default palette resolves to var(--chart-primary), var(--chart-secondary), var(--chart-tertiary), and var(--chart-error). Override those tokens on an ancestor (never --mat-sys-*) and the fills follow — no seriesColors in the config.
