Comparison Area Chart - Usage Examples

Overlaid, area-filled scenarios for comparing 2–3 competing mortgage loans. Each loan is a separate series (via seriesId); the low-opacity fills stay translucent so every band is readable through the ones on top. Areas are overlaid, not stacked — competing options don't sum to a whole.

1. Remaining Balance Over Term

Three loans on the same {{ comparisonPrincipal | currency: "USD" : "symbol" : "1.0-0" }} principal. createComparisonAreaChartConfig() defaults showArea on, curveType: 'monotone', points off, and the legend on. Pass seriesColors to give each scenario a distinct band. 

data = [
  ...amortize('30-yr @ 6.5%', 400000, 6.5, 30, 'balance'),
  ...amortize('15-yr @ 5.75%', 400000, 5.75, 15, 'balance'),
  ...amortize('30-yr refi @ 6.0%', 400000, 6.0, 30, 'balance'),
];

config = createComparisonAreaChartConfig({{ "{" }}
  data,
  seriesColors: ['#2196F3', '#4CAF50', '#FF9800'],
  showXAxis: true,
  showYAxis: true,
  xAxisLabel: 'Year',
  yAxisLabel: 'Remaining balance',
{{ "}" }});

2. Cumulative Interest Paid

The same three loans, switched to cumulative interest. Remaining balance and cumulative interest are the two complementary views of a loan comparison — expose both so a borrower sees the trade-off: the 15-yr clears its balance soonest and pays the least total interest, while the 30-yr options keep the monthly cost low but cost more over the life of the loan. 

// Same loans, y = cumulative interest instead of remaining balance
config = createComparisonAreaChartConfig({{ "{" }}
  data: interestData,
  seriesColors: ['#2196F3', '#4CAF50', '#FF9800'],
  showXAxis: true,
  showYAxis: true,
  xAxisLabel: 'Year',
  yAxisLabel: 'Cumulative interest',
{{ "}" }});

3. Click to Select a Scenario

Provide an onClick callback and enable showPoints so each yearly point is clickable. Here it records which loan and year the borrower picked. 

config = createComparisonAreaChartConfig({{ "{" }}
  data,
  showPoints: true,
  onClick: (event) => {{ "{" }}
    select(event.data.seriesId, event.data.x, event.data.y);
  {{ "}" }},
{{ "}" }});

4. Dynamic Rates With Signals

Recommended pattern: keep the loan terms in a signal and rebuild the config inside a computed(). Re-rolling the rates recomputes each amortization schedule and the chart animates between states. Here seriesColors is omitted, so the series fall through to the theme palette. 

loans = signal([
  {{ "{" }} label: '30-yr', rate: 6.5, termYears: 30 {{ "}" }},
  {{ "{" }} label: '15-yr', rate: 5.75, termYears: 15 {{ "}" }},
]);

config = computed(() =>
  createComparisonAreaChartConfig({{ "{" }}
    data: loans().flatMap(l => amortize(l.label, 400000, l.rate, l.termYears, 'balance')),
  {{ "}" }})
);
