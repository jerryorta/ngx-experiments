Heatmap Chart - Usage Examples

Practical patterns for createHeatmapChartConfig(). Feed one cell per {{ "{" }} row, col, value {{ "}" }} pair — rows sit on the y band axis, columns on the x band axis. The default mark: 'cell' draws a color-encoded grid; mark: 'bubble' a size-encoded one. Color comes from a named scheme or, unset, the theme token ramp. A value: null marks an empty cell.

1. Basic Cell Heatmap

Simplest usage: pass one cell per row × column pair. The default mark: 'cell' fills each cell from the theme token ramp; the null South·Q4 cell renders empty and is excluded from the color domain. 

// one cell per row (region) × col (quarter) pair
data = [
  {{ "{" }} col: 'Q1', row: 'North', value: 42 {{ "}" }},
  {{ "{" }} col: 'Q4', row: 'South', value: null {{ "}" }}, // empty cell
  // ...
];

config = createHeatmapChartConfig({{ "{" }} data {{ "}" }});

<nge-chart [config]="config" />

2. Bubble Heatmap

mark: 'bubble' swaps the cell rectangles for circles whose radius is sqrt-scaled to the value (double-encoded with the ramp color). 

config = createHeatmapChartConfig({{ "{" }} data, mark: 'bubble' {{ "}" }});

3. Named Color Scheme

A named scheme (d3-scale-chromatic) overrides the theme token ramp — here the perceptually-uniform 'viridis'. 

config = createHeatmapChartConfig({{ "{" }} data, scheme: 'viridis' {{ "}" }});

4. Show Values & Empty Cells

showValues: true prints each cell's value in the cell; empty (null) cells stay blank. 

config = createHeatmapChartConfig({{ "{" }} data, scheme: 'blues', showValues: true {{ "}" }});

5. Custom Color Domain

domain: [min, max] pins the color scale instead of the data's own extent — so several heatmaps can share one scale. Here the ramp spans a fixed 0–100. 

config = createHeatmapChartConfig({{ "{" }}
  data,
  domain: [0, 100],
  scheme: 'ylOrRd',
  showValues: true,
{{ "}" }});

6. Click to Inspect a Cell

The onClick payload carries the whole cell ({{ "{" }} row, col, value {{ "}" }}), so you can drill into the clicked intersection. 

config = createHeatmapChartConfig({{ "{" }}
  data,
  onClick: (event) => inspect(event.data.row, event.data.col, event.data.value),
{{ "}" }});
Clicked: {{ lastClicked() }}

7. Dynamic Data With Signals

Recommended pattern: hold the grid in a signal() and rebuild the config inside a computed(). Re-rolling the grid recolors every cell against the new value extent. 

liveData = signal<NgeHeatmapDataPoint[]>(makeSalesGrid());

config = computed(() => createHeatmapChartConfig({{ "{" }} data: liveData(), showValues: true {{
            "}"
          }}));

<nge-chart [config]="config()" />

8. Tooltip on Hover

showTooltip: true shows a follow-mouse tooltip with the cell's row · col and value on hover. Override the content with tooltip.formatContent. 

config = createHeatmapChartConfig({{ "{" }} data, showTooltip: true {{ "}" }});
