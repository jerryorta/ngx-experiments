Pareto Chart - Usage Examples

A Pareto chart pairs descending bars (individual category values, left axis) with a cumulative percentage line (right axis) — the classic 80/20 view for spotting the vital few contributors.

1. Basic Pareto

Pass category values to createParetoChartConfig(). Bars sort descending and the cumulative % line climbs to 100% on the right-hand axis. 

data = [
  {{ "{" }} label: 'Scratches', value: 42 {{ "}" }},
  {{ "{" }} label: 'Dents', value: 30 {{ "}" }},
  // ...
];

config = createParetoChartConfig({{ "{" }}
  data: this.data,
{{ "}" }});

<nge-chart [config]="config" />

2. With Value Labels

Set showLabels: true to print each bar's value. 

config = createParetoChartConfig({{ "{" }}
  data: this.data,
  showLabels: true,
{{ "}" }});

3. Preserve Input Order

Set sort: false to keep the data's original order (bars are no longer forced descending). The cumulative line still tracks the running total in that order. 

config = createParetoChartConfig({{ "{" }}
  data: this.data,
  sort: false,
{{ "}" }});

4. Click Handling

Provide an onBarClick callback; the event carries the clicked bar's data point. 

config = createParetoChartConfig({{ "{" }}
  data: this.data,
  onBarClick: (event) => {{ "{" }}
    console.log('Clicked:', event.data.label, event.data.value);
  {{ "}" }},
{{ "}" }});
Last clicked: {{ lastClicked() }}

5. Custom Colors

Use barColor for a uniform bar fill and lineColor for the cumulative line. 

config = createParetoChartConfig({{ "{" }}
  data: this.data,
  barColor: '#4CAF50',
  lineColor: '#E53935',
{{ "}" }});

6. Dynamic Data with Signals

Recommended pattern: rebuild the config from a computed signal so the chart re-sorts and recomputes the cumulative line whenever the data changes. 

data = signal<NgeBarDataPoint[]>([...]);

config = computed(() =>
  createParetoChartConfig({{ "{" }}
    data: this.data(),
    showTooltip: true,
  {{ "}" }})
);

<nge-chart [config]="config()" />
