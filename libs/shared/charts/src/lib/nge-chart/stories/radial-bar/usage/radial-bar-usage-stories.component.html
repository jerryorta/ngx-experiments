Radial Bar - Usage Examples

Practical examples for the radial-bar (polar) layer. One radial primitive fans out across the catalog via two options: mark picks the shape ({{ "{" }} 'bar' | 'area' | 'cell' {{ "}" }}) and wedge picks how the circle's angle is divided. Geometry is self-computed from the container (center + radius) — the cartesian axes stay off.

1. Basic Radial Bar

Simplest way to create a radial bar using createRadialBarChartConfig(). Each datum draws one arc from the center out to radialScale(value); a small padAngle separates the bars. 

data = [
  {{ "{" }} label: 'Mon', value: 30 {{ "}" }},
  {{ "{" }} label: 'Tue', value: 55 {{ "}" }},
  // ...
];

config = createRadialBarChartConfig({{ "{" }}
  data: this.data,
  padAngle: 0.02,
{{ "}" }});

<nge-chart [config]="config" />

2. Polar Area / Nightingale Rose

With wedge: 'equal' and padAngle: 0 the bars become full contiguous wedges — the classic Nightingale rose / polar-area chart. Every category gets the same angular slot; only the radius encodes value. 

config = createRadialBarChartConfig({{ "{" }}
  data: this.data,
  mark: 'bar',
  wedge: 'equal',
  padAngle: 0, // contiguous wedges
{{ "}" }});

3. Coxcomb (Value-proportional Wedges)

wedge: 'value' makes each wedge's angular extent proportional to its value (d3.pie()-style), so the angle encodes magnitude alongside the radius — a coxcomb. 

config = createRadialBarChartConfig({{ "{" }}
  data: this.data,
  mark: 'bar',
  wedge: 'value', // angle also encodes value
{{ "}" }});

4. Radial Line + Area (Multi-series)

mark: 'area' draws a closed radial area + outline per seriesId group over the category band centers — a multi-series radial line / area. Series colors cycle the palette by series index. 

data = [
  {{ "{" }} label: 'Q1', seriesId: '2023', value: 30 {{ "}" }},
  {{ "{" }} label: 'Q1', seriesId: '2024', value: 45 {{ "}" }},
  // ...
];

config = createRadialBarChartConfig({{ "{" }}
  data: this.data,
  mark: 'area',
  innerRadius: 0.1,
{{ "}" }});

5. Circular Heat Map

mark: 'cell' lays out a 2D grid of arc cells — the angular label crossed with the radial band ring — whose fill opacity encodes value (from the theme cell.minOpacity floor up to 1). 

data = [
  {{ "{" }} label: 'Mon', band: 'AM', value: 100 {{ "}" }},
  {{ "{" }} label: 'Mon', band: 'PM', value: 40 {{ "}" }},
  // ...
];

config = createRadialBarChartConfig({{ "{" }}
  data: this.data,
  mark: 'cell',
  innerRadius: 0.2,
{{ "}" }});

6. Donut Hole (innerRadius ratio)

innerRadius is a ratio (0–1) of the self-computed outer radius, not pixels — so it stays resize-safe. 0.4 carves a center hole, pushing the bars outward. 

config = createRadialBarChartConfig({{ "{" }}
  data: this.data,
  innerRadius: 0.4, // donut hole
  padAngle: 0.02,
{{ "}" }});

7. Click Handling

Provide an onClick callback in the config. The event carries the full data point for the clicked arc, so you can react to a selection. 

config = createRadialBarChartConfig({{ "{" }}
  data: this.data,
  onClick: (event) => {{ "{" }}
    console.log('Clicked:', event.data.label, event.data.value);
  {{ "}" }},
{{ "}" }});
Last clicked: {{ lastClicked() }}

8. Dynamic Data with Signals

Drive the chart from a signal and rebuild the config with computed(). Re-rolling every value re-runs the radial scale — the arcs grow / shrink to their new radius, animated. 

dynamicData = signal<NgeRadialBarDataPoint[]>(this.data);

dynamicConfig = computed(() =>
  createRadialBarChartConfig({{ "{" }} data: this.dynamicData(), padAngle: 0.02 {{ "}" }})
);

<nge-chart [config]="dynamicConfig()" />
