Scatter Chart - Usage Examples

Practical examples showing common usage patterns for the unified chart component with scatter plots. Each point needs a numeric x and y; color and size are optional per point.

1. Basic Usage

Simplest way to create a scatter plot using createScatterChartConfig() 

data = [
  {{ "{" }} x: 12, y: 18 {{ "}" }},
  {{ "{" }} x: 18, y: 25 {{ "}" }},
  // ...
];

config = createScatterChartConfig({{ "{" }}
  data: this.data,
  tooltip: {{ "{" }} enabled: true {{ "}" }},
{{ "}" }});

<nge-chart [config]="config" />

2. Click Handling

Handle point clicks by providing an onClick callback in the config. The event contains the clicked data point. 

config = createScatterChartConfig({{ "{" }}
  data: this.data,
  onClick: (event) => {{ "{" }}
    console.log('Clicked:', event.data.x, event.data.y);
  {{ "}" }},
{{ "}" }});
Last clicked: {{ lastClickedPoint() }}

3. Dynamic Data with Signals

Recommended pattern: Use a computed signal to rebuild the config when data changes. This ensures the chart re-renders when data updates. 

// Signal holds the reactive data
data = signal<NgeScatterDataPoint[]>([...]);

// Computed signal rebuilds config when data changes
config = computed(() =>
  createScatterChartConfig({{ "{" }}
    data: this.data(),
    tooltip: {{ "{" }} enabled: true {{ "}" }},
  {{ "}" }})
);

// Template uses the computed signal
<nge-chart [config]="config()" />

4. Per-Point Colors (Clusters)

Add a color property to individual data points to override the theme color. Useful for highlighting clusters or categorical groups. 

data = [
  {{ "{" }} x: 15, y: 20, color: '#F44336' {{ "}" }},
  {{ "{" }} x: 60, y: 65, color: '#2196F3' {{ "}" }},
  // ...
];

5. Per-Point Sizes (Bubble Style)

Add a size property (radius in px) to individual points to encode a third dimension, turning the scatter plot into a bubble chart. 

data = [
  {{ "{" }} x: 20, y: 30, size: 6 {{ "}" }},
  {{ "{" }} x: 50, y: 35, size: 16 {{ "}" }},
  {{ "{" }} x: 75, y: 72, size: 20 {{ "}" }},
  // ...
];

6. Tooltip on Hover

Enable tooltips by adding tooltip: {{ "{" }} enabled: true {{ "}" }}. Scatter uses Voronoi hit detection, so tooltips appear when hovering near a point, not only over it. The position option accepts 'above', 'below', or 'follow-mouse'. 

config = createScatterChartConfig({{ "{" }}
  data: this.data,
  tooltip: {{ "{" }}
    enabled: true,
    position: 'follow-mouse', // 'above' | 'below' | 'follow-mouse'
  {{ "}" }},
{{ "}" }});

7. Custom Tooltip Content

Provide formatContent to map each point to a custom {{ "{" }} label, value {{ "}" }}. Here raw x/y become "sqft" and "$K". 

config = createScatterChartConfig({{ "{" }}
  data: this.data,
  tooltip: {{ "{" }}
    enabled: true,
    formatContent: (point) => ({{ "{" }}
      label: `${{ "{" }}point.x.toLocaleString(){{ "}" }} sqft`,
      value: `$${{ "{" }}point.y{{ "}" }}K`,
    {{ "}" }}),
  {{ "}" }},
{{ "}" }});

8. Axis Labels & Domain Padding

Show axes with showXAxis/showYAxis, label them with xAxisLabel/yAxisLabel, and control breathing room around the data with xDomainPadding/yDomainPadding. Set yStartAtZero to anchor the Y axis at zero. 

config = createScatterChartConfig({{ "{" }}
  data: this.data,
  showXAxis: true,
  showYAxis: true,
  xAxisLabel: 'Square Feet',
  yAxisLabel: 'Price ($K)',
  xDomainPadding: 0.1,
  yDomainPadding: 0.15,
  yStartAtZero: true,
{{ "}" }});

9. Multi-Series with Legend

Group points into series by giving them a seriesId. Each series renders in a distinct palette color, and legend: {{ "{" }} enabled: true {{ "}" }} generates one legend entry per series (position defaults to 'bottom'). Because each point carries a seriesId, the default tooltip prefixes the series name on hover. 

data = [
  {{ "{" }} x: 12, y: 22, seriesId: 'Series A' {{ "}" }},
  {{ "{" }} x: 40, y: 48, seriesId: 'Series B' {{ "}" }},
  {{ "{" }} x: 66, y: 70, seriesId: 'Series C' {{ "}" }},
  // ...
];

config = createScatterChartConfig({{ "{" }}
  data: this.data,
  legend: {{ "{" }} enabled: true {{ "}" }},
  tooltip: {{ "{" }} enabled: true {{ "}" }},
{{ "}" }});

10. Per-Point Override within a Series

Series membership sets the default color, but a per-point color still wins over the series palette color, and a per-point size still sets that bubble's radius. Here one Series A point is forced red and one Series B point is enlarged, while the rest inherit their series color. 

data = [
  {{ "{" }} x: 14, y: 24, seriesId: 'Series A' {{ "}" }},
  // per-point color overrides the Series A palette color:
  {{ "{" }} x: 30, y: 28, seriesId: 'Series A', color: '#F44336' {{ "}" }},
  {{ "{" }} x: 46, y: 54, seriesId: 'Series B' {{ "}" }},
  // per-point size sets this bubble's radius (keeps the series color):
  {{ "{" }} x: 62, y: 58, seriesId: 'Series B', size: 20 {{ "}" }},
  // ...
];

11. Legend Series Selection (transform)

Click a legend entry below. The selected series stays prominent while the other series fade; clicking it again clears the selection. NgeScatterChartTransform is a plain class (no DI) that owns the interaction semantics — it turns legend clicks into a new config (fading via per-point opacity, which composes with the var(--chart-*) palette, unlike color math) and exposes it as a signal. It also supports programmatic axis zoom via setXDomain/setYDomain. 

// Component — plain class, no providers needed
transform = new NgeScatterChartTransform({{ "{" }}
  data: this.multiSeriesData, // points carry seriesId
  tooltip: {{ "{" }} enabled: true {{ "}" }},
{{ "}" }});

// Template — bind the derived config, wire legend clicks back in
<nge-chart
  [config]="transform.config()"
  (legendItemClick)="transform.onLegendItemClick($event)"
/>
