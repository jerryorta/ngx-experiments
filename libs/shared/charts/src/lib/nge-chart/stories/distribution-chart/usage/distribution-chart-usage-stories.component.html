Distribution Chart - Usage Examples

Practical patterns for createDistributionChartConfig(). Feed each category its RAW observations ({{ "{" }} category, values: number[] {{ "}" }}) — the layer computes the quartiles / KDE / jitter itself. One preset spans the family: render: 'box' (default) is a box-and-whisker, render: 'violin' a KDE density, and render: 'points' a strip / jitter / beeswarm point cloud.

1. Basic Box-and-Whisker

Simplest usage: pass raw values per category and let the layer summarise them. The box spans the IQR, the line marks the median, whiskers reach the 1.5·IQR fences, and dots are outliers. 

// raw observations per category — NOT pre-aggregated stats
data = [
  {{ "{" }} category: 'Class A', values: [76, 81, 68, /* ... */] {{ "}" }},
  {{ "{" }} category: 'Class B', values: [83, 79, 90, /* ... */] {{ "}" }},
];

config = createDistributionChartConfig({{ "{" }} data {{ "}" }});

<nge-chart [config]="config" />

2. Violin (KDE density)

render: 'violin' mirrors a kernel-density estimate around each category's axis so the width reads as local density; showInnerBox: true overlays a mini box-and-whisker for the quartiles. 

config = createDistributionChartConfig({{ "{" }}
  data,
  render: 'violin',
  showInnerBox: true,
{{ "}" }});

3. Beeswarm Point Cloud

render: 'points' plots every raw observation; jitter: 'beeswarm' packs them into a symmetric, non-overlapping swarm so density reads as swarm width. 

config = createDistributionChartConfig({{ "{" }} data, render: 'points', jitter: 'beeswarm' {{
            "}"
          }});

4. Jitter vs Strip

The same point cloud with two scatter strategies: jitter: 'uniform' spreads points randomly across the band width, while jitter: 'none' collapses them onto a single strip. 

uniform = createDistributionChartConfig({{ "{" }} data, render: 'points', jitter: 'uniform' {{
            "}"
          }});
strip   = createDistributionChartConfig({{ "{" }} data, render: 'points', jitter: 'none' {{
            "}"
          }});

5. Error Bars (±σ)

Drop the box with showBox: false for an error-bar plot; whiskerStat: 'stddev' sizes the whiskers to ±1 standard deviation and showMean: true marks the mean glyph. 

config = createDistributionChartConfig({{ "{" }}
  data,
  showBox: false,       // whiskers only — no IQR box
  whiskerStat: 'stddev', // ±1σ extent
  showMean: true,
{{ "}" }});

6. Horizontal Orientation

orientation: 'horizontal' swaps the scales — categories run down the y axis and the values read left-to-right. Ideal when category labels are long. 

config = createDistributionChartConfig({{ "{" }} data, orientation: 'horizontal' {{ "}" }});

7. Whisker Statistic (IQR vs min–max)

whiskerStat sets the whisker extent: 'iqr' draws Tukey 1.5·IQR fences and flags points beyond them as outliers, while 'minmax' stretches the whiskers all the way to the data extremes (no outliers). 

iqr    = createDistributionChartConfig({{ "{" }} data, whiskerStat: 'iqr' {{ "}" }});
minMax = createDistributionChartConfig({{ "{" }} data, whiskerStat: 'minmax' {{ "}" }});

8. Click to Inspect a Category

The onClick payload carries the whole distribution ({{ "{" }} category, values {{ "}" }}), so you can drill into the clicked category's raw observations. 

config = createDistributionChartConfig({{ "{" }}
  data,
  onClick: (event) => inspect(event.data.category, event.data.values),
{{ "}" }});
Clicked: {{ lastClicked() }}

9. Dynamic Data With Signals

Recommended pattern: hold the raw observations in a signal() and rebuild the config inside a computed(). Re-rolling the sample re-summarises every violin. 

liveData = signal<NgeDistributionDataPoint[]>(makeClasses());

config = computed(() => createDistributionChartConfig({{
            "{"
          }} data: liveData(), render: 'violin' {{ "}" }}));

<nge-chart [config]="config()" />
