Waterfall Chart - Theming Examples

Default Theme

No overrides — rise / fall / total pull from the theme's waterfall slice (semantic green rise, red fall, and a var(--chart-primary) total).

Muted Palette (riseColor / fallColor / totalColor)

The idiomatic way to recolor a waterfall: pass the semantic color options straight to the preset. Each colors the matching bar role.

Bold Palette

The same three options with high-saturation colors for a punchier read.

Monochrome

One hue at three weights — rise / fall / total as light / mid / dark slate.

Cumulative Overlay (cumulativeColor)

With cumulative: true, the running-total % line takes its color from cumulativeColor and renders against the secondary (right) axis.

Full Control via theme.waterfall

Compose a theme object instead of passing options — the waterfall slice styles the bar roles plus the step connector and value label.

Multiple Charts with Different Palettes

The same revenue bridge rendered with three color options sets. Each is the same <nge-chart> component with a different config.
