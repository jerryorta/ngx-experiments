Waterfall Chart - Usage Examples

Practical patterns for createWaterfallChartConfig(). Each datum is one bar: {{ "{" }} label, value {{ "}" }} deltas float on the running cumulative total (rise / fall colored), and a {{ "{" }} kind: 'total' {{ "}" }} datum is a subtotal / total column anchored at zero. Set cumulative: true for a Pareto-style running-total % overlay.

1. Basic Waterfall

Delta bars float from the prior running total to the new one — green for a gain, red for a loss — with step connectors bridging the bars. 

data = [
  {{ "{" }} label: 'Start',     value: 120 {{ "}" }},
  {{ "{" }} label: 'Product A', value:  45 {{ "}" }},
  {{ "{" }} label: 'Product B', value: -20 {{ "}" }},
  // ...
  {{ "{" }} label: 'Net', value: 0, kind: 'total' {{ "}" }},
];

config = createWaterfallChartConfig({{ "{" }} data {{ "}" }});

<nge-chart [config]="config" />

2. Subtotal / Total Bars

A kind: 'total' datum renders an anchored-at-zero checkpoint column at the current running total, without advancing it — e.g. a half-year (H1) and full-year total. 

data = [
  {{ "{" }} label: 'Start', value: 120 {{ "}" }},
  {{ "{" }} label: 'Q1',    value:  40 {{ "}" }},
  {{ "{" }} label: 'Q2',    value:  30 {{ "}" }},
  {{ "{" }} label: 'H1', value: 0, kind: 'total' {{ "}" }},
  // ...
];

3. Click to Inspect a Bar

Provide an onClick callback — each bar hands back its source datum (label, value, kind). 

config = createWaterfallChartConfig({{ "{" }}
  data,
  onClick: (event) => inspect(event.data.label, event.data.value),
{{ "}" }});
Clicked: {{ lastClicked() }}

4. Cumulative Overlay (Pareto-style)

cumulative: true overlays a running-total % line on a secondary (right) axis, reusing the line layer. For a classic Pareto (individual bars, sorted descending) reach for the dedicated createParetoChartConfig() preset instead. 

config = createWaterfallChartConfig({{ "{" }}
  data,
  cumulative: true, // adds the % line + right axis
{{ "}" }});

5. Dynamic Data With Signals

Recommended pattern: hold the data in a signal() and rebuild the config inside a computed(). Re-rolling the movements recomputes the config and the bars animate between states. 

liveData = signal<NgeWaterfallDataPoint[]>(buildData());

config = computed(() => createWaterfallChartConfig({{ "{" }} data: liveData() {{ "}" }}));

<nge-chart [config]="config()" />

6. Value Labels

showLabels prints each bar's value in place — a delta shows its signed movement, a total shows the running total. 

config = createWaterfallChartConfig({{ "{" }} data, showLabels: true {{ "}" }});
