Area Chart - Usage Examples

Practical patterns for createAreaChartConfig(). The fill is the primary mark: a single series fills from a zero baseline, multiple series can overlay or stack, and points carrying y0 become range bands. One stackOffset option switches between stacked, 100% stacked, streamgraph, and diverging layouts.

1. Plain Area (single series)

The simplest form — points with no seriesId form one series that fills from a zero baseline [0, y]. showLine strokes the top edge. 

data = [
  {{ "{" }} x: 'Jan', y: 24 {{ "}" }},
  {{ "{" }} x: 'Feb', y: 31 {{ "}" }},
  // ...
];

config = createAreaChartConfig({{ "{" }}
  data,
  fillOpacity: 0.3,
  showLine: true,
  showXAxis: true,
  showYAxis: true,
{{ "}" }});

2. Overlaid Multi-Series (non-summing)

Two independent series (each has a seriesId) with NO stackOffset — both rise from zero and overlap. Keep fillOpacity low so every band reads through the ones on top. Use this when the series don't sum to a whole. 

data = [
  {{ "{" }} seriesId: 'Desktop', x: 'Jan', y: 30 {{ "}" }},
  {{ "{" }} seriesId: 'Mobile',  x: 'Jan', y: 12 {{ "}" }},
  // ...
];

config = createAreaChartConfig({{ "{" }}
  data,
  fillOpacity: 0.25,
  legend: {{ "{" }} enabled: true {{ "}" }},
  seriesColors: ['#1E88E5', '#8E24AA'],
{{ "}" }});

3. Stacked Area (stackOffset: 'none')

With 2+ series and stackOffset: 'none', series stack from a zero baseline so the top edge is the running total. Ideal for parts that sum to a meaningful whole. 

config = createAreaChartConfig({{ "{" }}
  data: trafficData, // Organic + Paid + Referral
  legend: {{ "{" }} enabled: true {{ "}" }},
  stackOffset: 'none',
{{ "}" }});

4. 100% Stacked (stackOffset: 'expand')

stackOffset: 'expand' normalises every column to [0, 1] — each band shows its share of the total over time, regardless of the absolute magnitude. 

config = createAreaChartConfig({{ "{" }}
  data: trafficData,
  stackOffset: 'expand', // each column → 0…1
{{ "}" }});

5. Stream Graph (stackOffset: 'wiggle')

stackOffset: 'wiggle' centres the stack around a free-floating baseline for an organic streamgraph. The y axis is usually dropped since the baseline is no longer zero. 

config = createAreaChartConfig({{ "{" }}
  data: trafficData,
  stackOffset: 'wiggle',
  showXAxis: true, // y axis omitted
{{ "}" }});

6. Diverging (stackOffset: 'diverging')

stackOffset: 'diverging' stacks positive series up from zero and negative series down — best with mixed-sign data such as inflows vs. outflows. 

data = [
  {{ "{" }} seriesId: 'Inflows',  x: 'Jan', y:  40 {{ "}" }},
  {{ "{" }} seriesId: 'Outflows', x: 'Jan', y: -28 {{ "}" }},
  // ...
];

config = createAreaChartConfig({{ "{" }}
  data,
  stackOffset: 'diverging',
{{ "}" }});

7. Range Band (y0)

When points carry y0, the band spans [y0, y] (range mode) instead of rising from zero — perfect for min/max ranges or confidence intervals. showLine traces the upper edge. 

data = [
  {{ "{" }} x: 'Jan', y0: 2, y: 9 {{ "}" }},  // low → high
  {{ "{" }} x: 'Feb', y0: 3, y: 11 {{ "}" }},
  // ...
];

config = createAreaChartConfig({{ "{" }} data, showLine: true {{ "}" }});

8. Click to Inspect a Point

Provide an onClick callback — invisible per-vertex targets make each point clickable and hand back the source datum (seriesId, x, y). 

config = createAreaChartConfig({{ "{" }}
  data: trafficData,
  onClick: (event) => {{ "{" }}
    inspect(event.data.seriesId, event.data.x, event.data.y);
  {{ "}" }},
{{ "}" }});

9. Dynamic Data With Signals

Recommended pattern: hold the data in a signal() and rebuild the config inside a computed(). Re-rolling the values recomputes the config and the area animates between states. 

liveData = signal(buildData());

config = computed(() =>
  createAreaChartConfig({{ "{" }} data: liveData(), showLine: true {{ "}" }})
);
