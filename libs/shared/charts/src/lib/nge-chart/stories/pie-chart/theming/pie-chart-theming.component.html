Pie Chart - Theming Examples

Slice colours resolve per-datum color → the layer's seriesColors → the theme slice.colors palette (by input index). Override theme.pie.slice.colors to reskin the whole cycle, or set stroke / strokeWidth / opacity for separators and transparency. Light / dark is handled by the review container.

Default Theme

No theme provided — uses the built-in --chart-* token palette, which adapts to the surrounding light / dark surface.

Palette Overrides

Provide a custom theme.pie.slice.colors array. Slice input index maps to colors[index % length], so the palette cycles for any slice count.

Per-datum Colors

Set color on each data point to assign semantic colours. A per-datum color wins over the seriesColors / theme palette for that slice. 

data = [
  {{ "{" }} label: 'Rent', value: 1800, color: '#5C6BC0' {{ "}" }},
  {{ "{" }} label: 'Food', value: 600, color: '#66BB6A' {{ "}" }},
  // ...
];

Stroke Separators

A thicker slice.stroke / strokeWidth cleanly separates adjacent wedges — useful on dense pies or when slices share similar hues. 

theme: {{ "{" }}
  pie: {{ "{" }} slice: {{ "{" }} stroke: '#ffffff', strokeWidth: 3 {{ "}" }} {{ "}" }},
{{ "}" }}

Translucent Slices

Lower slice.opacity for a softer, layered look. 

theme: {{ "{" }}
  pie: {{ "{" }} slice: {{ "{" }} opacity: 0.7 {{ "}" }} {{ "}" }},
{{ "}" }}
