Pie Chart - Usage Examples

Practical examples for the pie / donut / semi-circle layer. One radial primitive fans out across three shapes via config: a full pie (innerRadius: 0), a donut (innerRadius as a ratio 0–1 carves the center hole), and a semi-circle / gauge (startAngle / endAngle sweep in radians). Each datum's value maps to a proportional arc.

1. Basic Pie

Simplest way to create a pie chart using createPieChartConfig(). Provide an array of {{ "{" }} label, value {{ "}" }} data points — one slice per point, in input order. 

data: NgePieDataPoint[] = [
  {{ "{" }} label: 'Rent', value: 1800 {{ "}" }},
  {{ "{" }} label: 'Food', value: 600 {{ "}" }},
  {{ "{" }} label: 'Transit', value: 300 {{ "}" }},
  {{ "{" }} label: 'Utilities', value: 250 {{ "}" }},
  {{ "{" }} label: 'Savings', value: 450 {{ "}" }},
];

config = createPieChartConfig({{ "{" }} data: this.data {{ "}" }});

<nge-chart [config]="config" />

2. Donut

innerRadius is a ratio (0–1) of the self-computed outer radius, not pixels — so it stays resize-safe. 0.6 carves a center hole 60% of the radius. 

config = createPieChartConfig({{ "{" }}
  data: this.data,
  innerRadius: 0.6, // donut
{{ "}" }});

3. Semi-circle Donut (Gauge)

startAngle and endAngle are in radians. Sweeping less than a full turn produces a semi-circle / gauge. Here -π/2 → π/2 draws the top half. 

config = createPieChartConfig({{ "{" }}
  data: this.data,
  innerRadius: 0.5,
  startAngle: -Math.PI / 2,
  endAngle: Math.PI / 2,
{{ "}" }});

4. Custom Slice Colors

Pass a seriesColors palette — slice input index maps to colors[index % length]. A per-datum color on any data point wins over the palette for that one slice. 

config = createPieChartConfig({{ "{" }}
  data: this.data,
  seriesColors: ['#1E88E5', '#43A047', '#FB8C00', '#8E24AA', '#00ACC1'],
{{ "}" }});

// Or override a single slice on the datum:
// {{ "{" }} label: 'Savings', value: 450, color: '#00ACC1' {{ "}" }}

5. Tooltip on Hover

Enable tooltips with tooltip: {{ "{" }} enabled: true {{ "}" }}. The default formatter shows the slice label and raw value; hover a slice to see it. 

config = createPieChartConfig({{ "{" }}
  data: this.data,
  innerRadius: 0.5,
  tooltip: {{ "{" }} enabled: true {{ "}" }},
{{ "}" }});

6. Click Handling

Provide an onClick callback in the config. The event carries the full data point for the clicked slice. 

config = createPieChartConfig({{ "{" }}
  data: this.data,
  onClick: (event) => {{ "{" }}
    console.log('Clicked:', event.data.label, event.data.value);
  {{ "}" }},
{{ "}" }});
Last clicked: {{ lastClicked() }}

7. With a Legend

The pie layer is inherently categorical — pair it with extractPieChartLegendItems(data) to build a legend whose swatches match the slices exactly. Combined here with a computed signal config so the legend + slices rebuild together when the data changes. 

legendConfig = computed(() => ({{ "{" }}
  ...createPieChartConfig({{ "{" }} data: this.data(), innerRadius: 0.55 {{ "}" }}),
  legend: {{ "{" }}
    enabled: true,
    items: extractPieChartLegendItems(this.data()),
    position: 'right',
    swatchShape: 'square',
  {{ "}" }},
{{ "}" }}));

<nge-chart [config]="legendConfig()" />
