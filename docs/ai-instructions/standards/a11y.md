title: Accessibility selectors (aria-label / data-testid)
---

- `aria-label` REQUIRED on: icon-only buttons/links, `<mat-button-toggle-group>`, overlay/menu triggers, inputs without a visible label. NOT on: buttons with visible text, inputs with a `<mat-label>`, already-labelled elements, `aria-hidden` decorative icons.
  - Static when fixed: `aria-label="Close dialog"` · Dynamic: `[attr.aria-label]="'Remove ' + item.name"`
  - Imperative + context-specific, 2–4 words ("Archive lead", not "Archive"); match `matTooltip` when present.
- `data-testid` only on non-semantic containers role/label can't target: app layout, sidenav, page-content wrappers, `<form>`, list/table wrappers, dialog areas. NOT on elements reachable by role+label, nor per-row/item.
  - kebab-case, `{feature}-{element-type}` (e.g. `lead-list-container`).
- Playwright locator priority: `getByRole({ name })` → `getByLabel` → `getByText` → `getByTestId` (containers only).
